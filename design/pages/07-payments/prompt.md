# Payments Prompt

<!-- markdownlint-disable MD013 MD012 -->

```text
Design a Payments page for Synapse.

Use `images/payments-v1.png` as the visual reference when available.

Show Filecoin Pay account state for the selected network:
- available balance
- funds
- lockup
- lockup rate
- storage runway
- token approval status
- deposit action
- approve action
- payment rails list

Make Mainnet/Calibration context visible. The page should clearly show when the user is on Calibration versus Mainnet.

Lead with user-facing storage balance, available funds, required top-up, and readiness. Keep payment rail internals in a details or advanced section.

The page should help the user understand whether uploads can proceed now. Payment rails, lockup, and approvals should support that readiness story, not dominate it.
```
